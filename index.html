OSI and TCP/IP Models

OSI framework standardizes communication processes between networked systems.

The 7 Layers of the OSI

1)Physical Layer: Transfers raw data (bits) over physical mediums (Ethernet, Wi-Fi).

2) Data Link Layer: Handles MAC addresses and error detection (Switches, ARP).

3)Network Layer: Manages IP addressing and routing (IP, Routers).

4)Transport Layer: Ensures reliable (TCP) or fast (UDP) data delivery (TCP, UDP).

5)Session Layer: Establishes and maintains communication sessions (NetBIOS, RPC).

6)Presentation Layer: Formats data for the application layer (SSL, JPEG).

7)Application Layer: Interfaces directly with users via PROTOCOLS (HTTP, FTP, DNS).

The TCP/IP Model is a practical 4-layer framework for internet PROTOCOLS and real-world networking.

1)Link Layer: Combines OSI's Physical and Data Link layers for hardware communication (Ethernet, Wi-Fi).

2)Internet Layer: Handles logical addressing and routing (IP, ICMP).

3)Transport Layer: Ensures data delivery using TCP/UDP (TCP, UDP)

4)Application Layer: Includes PROTOCOLS for user applications (HTTP, FTP, DNS).

Mac Adress

A MAC address is a unique 48-bit (6-byte) identifier for a device's NIC, used for LAN communication. Displayed in hexadecimal (e.g., 00:1A:2B:3C:4D:5E), it ensures data reaches the correct device.

MAC Address Components:

1)OUI (Organizationally Unique Identifier) First 24 bits (e.g., 00:1A:2B), assigned to the manufacturer by IEEE

2)Device Identifier(vendor specific) Last 24 bits (e.g., 3C:4D:5E), uniquely identifies the device.

Topologies

1)Bus. . DESCRIPTION: Devices share a single cable; data travels in both directions. PROTOCOLS: Ethernet, CSMA/CD; EXAMPLE: Early Ethernet networks.

2)Star. DESCRIPTION: Devices connect to a central hub or switch; easy to troubleshoot. PROTOCOLS: Ethernet, Wi-Fi, TCP/IP; EXAMPLE: Home or office networks.

3)Ring. DESCRIPTION: Devices are arranged in a circular loop; data flows in one or both directions. PROTOCOLS: Token Ring, FDDI; EXAMPLE: FDDI (Fiber Distributed Data Interface).

4)Mesh. DESCRIPTION:Devices interconnect directly, ensuring redundancy and reliability. PROTOCOLS: OSPF, BGP; EXAMPLE: Internet backbone networks.

5)Tree. DESCRIPTION: Hierarchical combination of star and bus topologies; scalable for large setups. PROTOCOLS: Ethernet, TCP/IP; EXAMPLE: Corporate networks..

6)Hybrid. DESCRIPTION: Combines multiple topology types for flexibility. PROTOCOLS:Varies (depends on used topologies); EXAMPLE: WANs in organizations.

7)Point-to-Point DESCRIPTION: Direct connection between two devices. PROTOCOLS: PPP, HDLC; EXAMPLE: Phone line communication.

TCP vs UDP: Key Differences

1)Connections-
TCP: Connection-oriented (establishes a session).
UDP: Connectionless (no session established).

2)Reliability-
TCP: Ensures reliable data delivery with error checking.
UDP: No guarantee of delivery or error checking.

3)Speed-
TCP: Slower due to reliability mechanisms.
UDP: Faster as it skips reliability checks.

4)Use Cases-
TCP: File transfer, web browsing (e.g., HTTP, FTP).
UDP: Streaming, gaming, VoIP (e.g., DNS, RTP).

5)Header Size -
TCP: Larger (20 bytes).
UDP: Smaller (8 bytes).

VLAN and Broadcast Domain

VLAN-Logical network segmentation to isolate traffic and improve security.

Broadcast Domain-VLANs limit the area where broadcast packets are sent and received.

VLAN Ranges-- Normal: 1–1005 | Extended: 1006–4094

Default VLAN-VLAN 1 (default for all ports).

Native VLAN-Handles untagged traffic on trunk ports (default is VLAN 1 but can be changed).

VLAN Configuration Commands

Create a VLAN- vlan < VLAN_ID>

Name a VLAN- vlan < VLAN_NAME>

Assign VLAN to an interface- interface < INTERFACE>, switchport mode access, switchport access vlan < VLAN_ID>,

Show VLANs- show vlan brief

Delete a VLAN- no vlan < VLAN_ID>

Switch and Router Modes

User EXEC Mode(Limited commands for basic monitoring.): Switch> or Router>

Privileged EXEC Mode(Full access to view and manage configuration.) Switch# or Router#

Global Config Mode Switch(config)#

Interface Config Mode Switch(config-if)#

Switch Interface Configuration Mode Commands

Enter Interface Config Mode- interface < INTERFACE>

Set Port as Access Mode- switchport mode access

Assign VLAN to Port-switchport access vlan < VLAN_ID>

Set Port as Trunk Mode-switchport mode trunk

Set Native VLAN on Trunk- switchport trunk native vlan < VLAN_ID>

Enable Port-no shutdown

Disable Port-shutdown

Show Interface Config- show running-config interface < INTERFACE>

EXAMPLE:interface FastEthernet0/1; switchport mode access; switchport access vlan 10; no shutdown;

Configuration Commands for Inter-VLAN Routing

Enable Routing on the Router: interface < router_interface> no shutdown

Create Subinterfaces for VLANs: interface < router_interface>.< VLAN_ID> encapsulation dot1Q < VLAN_ID> ip address < IP_ADDRESS> < SUBNET_MASK>

Example for VLAN 10: interface GigabitEthernet0/0.10 encapsulation dot1Q 10 ip address 192.168.10.1 255.255.255.0

Option 2: Layer 3 Switch

Enable IP Routing:ip routing

Create VLAN Interfaces (SVIs): interface vlan < VLAN_ID> ip address < IP_ADDRESS> < SUBNET_MASK> no shutdown

Verification Commands: Check VLANs: show vlan brief

Verify Routing: show ip route

Test Connectivity: ping < IP_ADDRESS>
